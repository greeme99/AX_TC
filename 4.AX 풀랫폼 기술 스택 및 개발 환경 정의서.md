# **🛠️ AX Control Tower: 기술 스택 및 개발 환경 정의서**

본 문서는 '가벼움, 보안, 확장성'을 핵심 가치로 하는 AX 플랫폼의 기술 표준을 정의하며, 구글 Project IDX 및 Antigravity 개발 환경에 최적화되어 있습니다.

## **1\. Core Tech Stack (기술 스택)**

### **🎨 Frontend (사용자 인터페이스)**

* **Framework:** **React 18+ (Vite 기반)**  
  * *선정이유:* 가볍고 빠른 빌드 속도(Vite)와 Project IDX의 기본 템플릿과의 높은 호환성.  
* **Styling:** **Tailwind CSS**  
  * *선정이유:* 별도의 CSS 파일 관리 없이 클래스 기반으로 즉시 UI 구현이 가능하며, 코드의 경량화 유지.  
* **Icon Library:** **Lucide React**  
  * *선정이유:* 깔끔하고 전문적인 인터페이스를 위한 고품질의 가벼운 아이콘 세트.  
* **State Management:** **React Context API / Signals**  
  * *선정이유:* 추가 라이브러리 설치를 최소화하면서 플랫폼 전체의 인증 및 보안 상태를 관리.

### **⚙️ Backend & Infrastructure (인프라)**

* **Platform:** **Google Firebase (Project IDX 통합)**  
  * **Authentication:** Firebase Auth (구글 계정 연동 및 사내 이메일 인증)  
  * **Database:** Cloud Firestore (실시간 과제 현황 및 프롬프트 템플릿 저장)  
  * **Storage:** Firebase Storage (교육 자료 업로드 및 보안 지침서 보관)  
  * **Hosting:** Firebase Hosting (원클릭 배포 및 HTTPS 보안 적용)

## **💻 2\. 개발 환경 설정 (Development Environment)**

* **IDE:** **Google Project IDX** (브라우저 기반 클라우드 개발 환경)  
* **Version Control:** **GitHub** (사내 전용 Private Repository)  
* **Build Tool:** **Vite** (초고속 HMR 지원으로 개발 생산성 극대화)  
* **Environment Config:** .env 파일을 활용한 API Key 및 접속 정보 분리 관리.

## **🛡️ 3\. 보안 및 운영 표준 (Security Standards)**

1. **Zero-Hardcoding:** 모든 외부 API Key(Gemini 등)는 IDX 환경 변수 또는 Firebase Secrets에 등록하여 코드 유출 방지.  
2. **API Proxying:** AI 모델 호출 시 클라이언트에서 직접 호출하지 않고, 서버리스 함수(Firebase Functions)를 경유하도록 설계하여 인증 정보 노출 차단.  
3. **Client-Side Filter:** filterSensitiveData 함수를 구현하여 개인정보나 기밀 키워드가 포함된 프롬프트는 전송 전 1차 차단.  
4. **Audit Logging:** 모든 AI 사용 이력을 Firestore의 로그 컬렉션에 자동 기록하여 사후 점검 체계 마련.

## **🚀 4\. 배포 및 공유 프로세스**

1. **Local Dev:** IDX 클라우드 서버에서 실시간 미리보기를 통해 개발.  
2. **Staging:** firebase hosting:channel deploy를 통해 테스트용 URL 생성 및 관계자 검수.  
3. **Production:** firebase deploy를 실행하여 정식 도메인에 배포.

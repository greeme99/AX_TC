# **🛠️ AX Control Tower 개발 명세서 (Security-First Edition)**

본 명세서는 플랫폼 개발 시 반드시 준수해야 할 기술적 표준과 보안 구현 방식을 정의합니다. 개발자는 본 내용을 숙지하고 모든 기능 구현의 기초로 삼아야 합니다.

## **🏗️ 1\. 시스템 아키텍처 및 환경 구성**

### **1.1 환경 변수 관리 (Environment Secrets)**

* **원칙:** 인증 정보, API 키, DB 접속 정보는 절대로 소스코드에 하드코딩하지 않습니다.  
* **구현:** \* .env 파일을 사용하여 로컬 환경을 관리하고, 운영 환경에서는 시스템 키체인 또는 Secret Manager를 사용합니다.  
  * .gitignore에 .env 파일을 반드시 포함하여 버전 관리 시스템(Git)에 유출되지 않도록 합니다.

### **1.2 서버 및 데이터베이스 권한**

* **최소 권한 원칙:** 모든 서비스 계정은 해당 기능 수행에 필요한 최소한의 권한만 가집니다.  
  * 예: 단순 조회용 API 계정은 SELECT 권한만 부여.

## **🛡️ 2\. 핵심 보안 기능 구현 명세**

### **2.1 인증 및 인가 (Authentication & Authorization)**

* **계정 분리:** 소스코드 및 AI 대화창 내에 ID/PW를 포함하지 않도록 프론트엔드와 백엔드 로직을 분리합니다.  
* **API Key 보호:** 외부 AI 모델(Gemini, GPT 등) 호출 시 사용되는 API Key는 백엔드 서버 측 환경변수로만 관리하며, 클라이언트(브라우저)에 노출되지 않도록 프록시 서버를 경유합니다.

### **2.2 코드 및 데이터 자가 진단 엔진 (Security Scanner)**

* **구현 로직:** 사용자가 업로드한 코드나 프롬프트를 분석하여 아래 4대 항목을 체크하는 정규표현식 및 AI 분석 레이어를 구축합니다.  
  1. **하드코딩:** API Key, ID/PW 패턴 매칭 검사.  
  2. **입력값 검증:** SQL Injection, XSS 공격 패턴 포함 여부.  
  3. **권한 통제:** 불필요한 관리자 권한 요청 로직 탐지.  
  4. **데이터 공유:** 개인정보(이메일, 전화번호) 및 영업비밀 키워드 탐지.

## **✉️ 3\. 기능별 세부 개발 표준**

### **3.1 알림 및 메일링 시스템**

* **자격 증명:** SMTP 계정 정보는 외부 시스템 설정값으로 관리합니다.  
* **데이터 참조:** 수신자 목록은 하드코딩하지 않고 암호화된 DB 테이블에서 호출합니다.  
* **로그 관리:** 발송 이력은 로컬 DB에 기록하되, 메일 본문의 기밀 정보가 외부 AI API로 재전송되지 않도록 필터링 로직을 적용합니다.

### **3.2 데이터베이스 연동 및 쿼리**

* **가명화 처리:** AI를 이용해 동적 쿼리를 생성하거나 디버깅할 경우, 실제 테이블명(예: user\_salary) 대신 가명(예: tbl\_A)을 사용하여 프롬프트를 구성합니다.  
* **샘플 데이터 사용:** 개발 및 테스트 단계에서는 실제 운영 데이터를 절대 사용하지 않으며, 무작위로 생성된 더미 데이터를 사용합니다.

### **3.3 파일 업로드 및 API 연동**

* **업로드 보안:** 파일 업로드 시 확장자(White-list 방식), 파일 크기 제한을 서버 측에서 강제합니다.  
* **외부 전송 통제:** 내부 데이터를 외부 서버로 전송하는 모든 코드는 별도의 '보안 검토 로그'를 남기고 관리자 승인 플래그가 있어야 활성화되도록 구현합니다.

## **🤖 4\. AI 도구 활용 가이드라인 (DevOps)**

### **4.1 코딩 AI (Cursor, Copilot) 활용**

* **정보 유출 방지:** .cursorignore 또는 .copilotignore 파일을 프로젝트 루트에 생성하여 인증 정보가 포함된 파일이 AI 서버로 전송되지 않도록 설정합니다.  
* **코드 검증:** AI가 제안한 코드는 반드시 사람이 보안 취약점(SQL Injection 등)을 검토한 후 커밋(Commit)합니다.

### **4.2 대화형 AI (ChatGPT, Claude) 활용**

* **데이터 학습 제외:** 기업용 플랜(Team/Enterprise) 여부를 확인하고, 설정에서 '학습 제외(Opt-out)' 처리를 확인합니다.  
* **공유 금지:** 대화 내용 공유(Share Link) 기능을 사용할 때는 내부 기밀 정보가 포함되지 않았는지 더블 체크하는 절차를 가집니다.

## **📈 5\. 사용 이력 및 모니터링**

* **Audit Log:** 모든 AI Tool 사용 내역(입력 프롬프트 포함)은 팀 단위로 암호화되어 기록 및 보관되어야 합니다.  
* **이상 징후 알림:** 단시간 내 대량의 기밀 키워드가 포함된 프롬프트가 입력될 경우 관리자에게 즉시 알림을 발송합니다.
